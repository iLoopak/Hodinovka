# Android APK build (Capacitor)

Hodinovka is a 100% client-side static PWA. Capacitor wraps the Next.js static
export (`out/`) in a native Android WebView â€” there is no server component. On
the native shell, PDF **sharing** goes through the Capacitor Share plugin and
**saving** through the Filesystem plugin (see `lib/native.ts`); the web/PWA build
keeps its Web Share API + `<a download>` + `mailto:` behaviour unchanged.

## Prerequisites

- **JDK** (17 or 21).
- **Android SDK.** Gradle finds it via `ANDROID_HOME` / `ANDROID_SDK_ROOT`, or via
  `android/local.properties` (`sdk.dir=/path/to/Android/sdk`) â€” this file is
  auto-generated by Android Studio and is git-ignored (machine-specific).
- Optionally **Android Studio** for an emulator / signing / Play Store upload.

## What is and isn't committed

- **Committed:** the `android/` native project + `capacitor.config.ts`.
- **Git-ignored (regenerated by `cap sync`):** copied web assets
  (`android/app/src/main/assets/public`), the generated Capacitor config/plugin
  files, `local.properties`, and all Gradle build output (`android/**/build/`,
  `.gradle/`, `*.apk`).

So after a fresh clone you must run a sync before the native project is buildable.

## Commands

```bash
# Rebuild the web export and copy it into the native project
npm run android:sync

# Open the project in Android Studio (run / debug / emulator)
npm run android:open

# One-shot debug APK: build export â†’ sync â†’ Gradle assembleDebug
npm run android:apk
# â†’ android/app/build/outputs/apk/debug/app-debug.apk
```

`android:sync` / `android:apk` run `next build` first, so the APK always bundles
the latest web assets. If you edit only native code, `npx cap sync android` alone
is enough.

## App identity

- **applicationId / namespace:** `com.iloopak.hodinovka` (permanent package id)
- **App label:** Hodinovka (`android/app/src/main/res/values/strings.xml`)

## Release build (later)

`assembleDebug` produces an unsigned-for-store debug APK â€” fine for sideloading
and on-device testing. For Play Store you'll need a release keystore and
`assembleRelease` / `bundleRelease` (AAB); wire the signing config in
`android/app/build.gradle`. Keep the keystore **out** of git.
